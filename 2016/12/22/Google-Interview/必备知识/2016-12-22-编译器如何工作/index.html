编译器如何工作

发表于 2016-12-22   |   分类于 google-interview   |     |   阅读次数   |  

基本概念

Lexer (词法分析器): source -> tokens(语元)
Parser(语法解析器) tokens -> AST(抽象语法树)
Generator AST -> Target Code



Lexer (词法分析器)

要实现一门语言，第一要务就是要能够处理文本文件，搞明白其中究竟写了些什么。传统上，我们会先利用“词法分析器”（也称为“扫描器”）将输入切成“语元（token）”，然后再做处理。词法分析器返回的每个语元都带有一个语元编号，此外可能还会附带一些元数据（比如某个数值）。

每个语言都有自己的词法分析器的实现.

Parser(语法解析器)

抽象语法树(AST)作用在于牢牢抓住程序的脉络，从而方便编译过程的后续环节（如代码生成）对程序进行解读。AST就是开发者为语言量身定制的一套模型，基本上语言中的每种结构都与一种AST对象相对应。

开始构造AST之前，先要准备好用于构造AST的语法解析器。说白了，就是要利用语法解析器把“x+y”这样的输入（由词法分析器返回的三个语元）分解成由代码生成的AST.
