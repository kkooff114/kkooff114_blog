[TOC]

0101. 活动列表读取

请求Url:

入参:

字段 描述 说明
openId 微信用户编码 通过中间表转换成customerCode
promotionStatus 促销协议状态 编码见数据字典
rebatePattern 返还方式编码 编码见数据字典

出参:

字段 描述 来源表 说明
agreementId 协议编号 1
createDate 创建日期 1
customerCode 客户编号 1
customerName 客户名称 6 取数说明4
endDate 协议结束时间yyyy-MM-dd 1
modifiedDate 修改日期 1
month 月 2,3,4 多个”,”分隔, 取数说明2
promotionId 活动id 1
rebatePattern 返还方式编码 1 编码见数据字典
salesId 业代编号 1
startDate 协议开始时间yyyy-MM-dd 1
tpApplicationId 活动申请编号 1
tpName 活动名称 5 取数说明1
year 年 2,3,4 取数说明3

涉及相关表:

  1. CPM_PROMOT_AGREEMNT_HEADER (促销协议抬头表)
  2. CPM_PROMOT_AGREEMNT_CONTROL_CARD (促销协议-内容控制-券)
  3. CPM_PROMOT_AGREEMNT_CONTROL_MONEY (促销协议-内容控制-现金)
  4. CPM_PROMOT_AGREEMNT_CONTROL_SKU (促销协议-内容控制-SKU)
  5. CPM_PROMOT_HEADER (促销活动抬头表)
  6. CPM_CRM_CUSTOMER_DATA (CRM终端客户表)

接口逻辑:

  1. 到表1中根据入参customerCode查找返还结果集
  2. 入参promotionStatus为””, 返回所有
  3. 入参promotionStatus为”PS01”:
    1. 如表1.rebatePattern == RT01, 去表4中查找满足t_month==当前月的协议, 返回;
    2. 如表1.rebatePattern == RT02, 去表2中查找满足t_month==当前月的协议, 返回;
    3. 如表1.rebatePattern == RT03, 去表3中查找满足t_month==当前月的协议, 返回
  4. 入参promotionStatus为”PS02”:
    1. 如表1.rebatePattern == RT01, 去表4中查找满足所有t_month<当前月的协议, 返回;
    2. 如表1.rebatePattern == RT02, 去表2中查找满足所有t_month<当前月的协议, 返回;
    3. 如表1.rebatePattern == RT03, 去表3中查找满足所有t_month<当前月的协议, 返回;
  5. 入参promotionStatus为”PS03”:
    1. 如表1.rebatePattern == RT01, 去表4中查找满足所有t_month>当前月的协议, 返回;
    2. 如表1.rebatePattern == RT02, 去表2中查找满足所有t_month>当前月的协议, 返回;
    3. 如表1.rebatePattern == RT03, 去表3中查找满足所有t_month>当前月的协议, 返回;
  6. 三个入参取交集

字段取数说明:

  1. tpName:根据promotionId >> trade_promotion_id 返回 promotion_name
  2. month: 根据入参rebatePattern的取值
    1. 如RT01到表4中按agreementId查找多条结果, 返还t_month;
    2. 如RT02到表2中按agreementId查找多条结果, 返还t_month;
    3. 如RT03到表3中按agreementId查找多条结果, 返还t_month;
  3. year: 根据入参rebatePattern的取值
    1. 如RT01到表4中按agreementId查找多条结果, 返还t_year;
    2. 如RT02到表2中按agreementId查找多条结果, 返还t_year;
    3. 如RT03到表3中按agreementId查找多条结果, 返还t_year;
  4. customerName: 根据customerCode >> customer_id 返回 customer_name

0103. 电子协议确认提交

请求Url:

入参:

字段 描述 说明
agreementId 协议id
confirm Y/N

出参:
无

涉及相关表:

  1. CPM_PROMOT_AGREEMNT_HEADER (促销协议抬头表)

接口逻辑:

  1. 约定confirm入参 Y对应CS02; N对应CS03
  2. 根据agreementId 查找表1 , 判断最新一条comfirmStatus
    1. comfirmStatus == CS01 即待确认, 直接修改为confirm入参
    2. comfirmStatus == CS04, 入参confirm==Y. 直接修改confirmStatus为CS02, 并将倒数第二条记录打上del_flag
    3. comfirmStatus == CS04, 入参confirm==N. 直接修改confirmStatus为CS03

遗留问题:

  • 确认状态是否需要保存到合同中心

rebateDate 返还日期 7
sku sku 8 取数说明4

涉及相关表:

  1. CPM_PROMOT_AGREEMNT_HEADER (促销协议抬头表)
  2. CPM_PROMOT_HEADER (促销活动抬头表)
  3. CPM_CRM_CUSTOMER_DATA (CRM终端客户表)
  4. CPM_PROMOT_AGREEMNT_CONTROL_TYPE (促销协议-内容控制-促销形式)
  5. CPM_PROMOT_RETURN_CARD (促销返还记录表-卡券)
  6. CPM_PROMOT_RETURN_MONEY (促销返还记录表-现金)
  7. CPM_PROMOT_RETURN_SKU_HEADER (促销返还记录表-实物-抬头表)
  8. CPM_PROMOT_RETURN_SKU_LINER (促销返还记录表-实物-明细表)
  9. CPM_CRM_PRODUCT_UNIT_EXCHANGE (CRM产品-单位换算)
  10. CPM_CRM_PRODUCT_BASE_INFO (CRM产品基本信息表)
  11. CPM_PROMOT_AGREEMNT_ACTION_DISPLAY_MONTH (促销协议-执行-陈列内容形式-月份)

接口逻辑:

  1. 根据入参agreementId到表1中查找rebatePattern
  2. 实物(rebatePattern == RT01):
    1. 根据agreementId到表11 统计 如所有的result == "RS03"则算合格, 否则不合格
    2. 如不合格qualified == false返回; 否则继续
    3. 取表7中数据返回
  3. 卡券(rebatePattern == RT02):
    1. 根据agreementId到表11 统计 如所有的result == "RS03"则算合格, 否则不合格
    2. 如不合格qualified == false返回; 否则继续
    3. 取表5中数据返回
  4. 现金(rebatePattern == RT03):
    1. 根据agreementId到表11 统计 如所有的result == "RS03"则算合格, 否则不合格
    2. 如不合格qualified == false返回; 否则继续
    1. 取表6中数据返回

字段取数说明:

  1. tpName:根据promotionId >> trade_promotion_id 返回 promotion_name
  2. customerName: 根据customerCode >> customer_id 返回 customer_name
  3. displayCtreria:
    1. 根据agreementId到表4中查找结果集
    2. 将i中结果集dis_pattern到表???查找文本描述
    3. 返回displayName1+dis_ctreria1, displayName2+dis_ctreria2, 如: 地堆2层，冰箱2层
  4. sku:
    1. 根据agreementId到表7 中查找product_rebate_id
    2. 到表8中product_rebate_id >> product_rebate_id 返回 product_name + product_number + product_unit_desc, 如: 500ML怡宝1箱，500ml的魔力1箱
    3. product_name: 到表10中 根据product_id>> inventory_item_id 返回 product_description
    4. product_unit_desc: 到表9中 根据product_unit >> product_unit_of_measure 返回 product_unit_of_measure_text
  5. qualified: 执行合格标准, 判断条件确认???

遗留问题:

  • 商户现金返还支持微信和银行卡? 表CPM_PROMOT_RETURN_MONEY中 payChannel作用?
  • 陈列形式没有文本配置表, displayName没地方取
