请求Url:

入参:

字段 描述

字段 描述 来源表

  1. CPM_PROMOT_HEADER (促销活动抬头表)
  2. CPM_PROMOT_STRATEGY_CHECK (促销活动-营销策略核对)
  3. CPM_CRM_CUSTOMER_CHANNEL (CRM客户渠道表)
  4. CPM_PROMOT_DISPLAY_STANDARD (促销活动-陈列标准)
  5. CPM_CRM_PRODUCT_UNIT_EXCHANGE (CRM产品-单位换算)
  6. CPM_PROMOT_RETURN_TYPE (促销活动-返还形式)
  7. CPM_CRM_PRODUCT_BASE_INFO (CRM产品基本信息表)
  8. CPM_PROMOT_HEADER_DEALER (促销活动抬头表-经销商)
  9. CPM_CRM_DEALER_DISTRIBUTOR (CRM配送经销分销表)
  10. CPM_PROMOT_COST_ASCRIPTION (促销活动-费用归属)
  11. CPM_CRM_USER_BASE_INFO (CRM用户基本信息表)

接口逻辑:

  1. 到表11中根据入参username >> employee_number查找office_code与secondary_office
  2. 到表1中,用步骤1中的结果进行查找where (步骤1.office_code == 表1.office_code and 表1.secondary_office == NULL) or 步骤1.secondary_office in 表1.secondary_office
  3. 根据步骤2中返回的活动列表, 补全其他返回参数后, 返回

字段取数说明:

  1. channelDesc: 根据channelId >> customer_channel_id 返回 customer_channel_text
  2. unitDesc: 根据unitCode >> product_unit_of_measure 返回 product_unit_of_measure_text
  3. skuName: 根据skuId >> inventory_item_id 返回 product_description
  4. distributorName: 根据distributorId >> bussiness_partner_no 返回 customer_name

  • 表CPM_PROMOT_RETURN_TYPE中促销类型作用
  • 表CPM_PROMOT_RETURN_TYPE中缺少sku预计返还数量
  • 表CPM_PROMOT_RETURN_TYPE中返还单位作用
  • 费用归属在手机上的作用

字段 描述 存表 说明
dispayId 陈列形式编码 5
displayCtreria 执行标准 5
month 月份 5
promotionId 活动id 5

出参:

字段 描述 来源表
agreementId 协议id 1

涉及相关表:

  1. CPM_PROMOT_AGREEMNT_HEADER (促销协议抬头表)
  2. CPM_PROMOT_AGREEMNT_CONTROL_CARD (促销协议-内容控制-券)
  3. CPM_PROMOT_AGREEMNT_CONTROL_MONEY (促销协议-内容控制-现金)
  4. CPM_PROMOT_AGREEMNT_CONTROL_SKU (促销协议-内容控制-SKU)
  5. CPM_PROMOT_AGREEMNT_CONTROL_TYPE (促销协议-内容控制-促销形式)
  6. CPM_CRM_USER_BASE_INFO (CRM用户基本信息表)

接口逻辑:

  1. 到表1 中用入参promotionId, customerCode, !del_flag comfirmStatus != CS03查找, 如果命中见下三种情况, 否则进入步骤2:
    1. 若最新一条comfirmStatus == CS01, 返回agreementId status=0001 ,msg=该活动已有一条待确认的协议, 无需再次创建
    2. 若最新一条comfirmStatus == CS02, 返回agreementId status=0001 ,msg=该活动已签订协议, 无需再次创建
    3. 若最新一条comfirmStatus == CS04, 返回agreementId status=0001 ,msg=该活动已有一条待确认的补充协议, 无需再次创建
  2. 到表6中根据入参username >> employee_number查找region_code, large_zone_code, local_code, city_code, office_code与secondary_office
  3. 将入参及步骤1查找的协议相关字段存入相应的表, confirmStatus默认CS01
  4. 根据rebatePattern的取值: RT01存 controlSkutList; RT02 存controlCardList; RT03存controlCashList
  5. 调用合同中心接口???创建电子合同
  6. 调用微信推送协议接口

遗留问题:

  • 表CPM_PROMOT_AGREEMNT_HEADER中协议类型作用
  • 经销商到月的? CPM_PROMOT_AGREEMNT_CONTROL_CARD缺少经销商id
  • 经销商到月的? CPM_PROMOT_AGREEMNT_CONTROL_MONEY缺少经销商id
  • CPM_PROMOT_AGREEMNT_CONTROL_SKU缺少单位id
  • 未整理合同中心创建电子合同接口
  • 未整理微信推送协议接口

接口逻辑:

  1. 到表6中根据入参username >> employee_number查找secondary_office
  2. 到表1中使用入参agreementId, customerCode 及 步骤1中的secondary_office 查找结果集.
    1. 有del_flag 标记的不返回
    2. 如最新一条confirmStatus==CS01 || CS02, 返回本条记录
    3. 如最新一条confirmStatus==CS04, 倒数第二条confirmStatus==CS02, 本条记录不返还, 返回倒数第二条, 并将返回的倒数第二条协议的confirmStatus填充CS04
    4. 如最新一条confirmStatus==CS04, 倒数第二条confirmStatus==CS02, 返回本条记录
    5. 如最新一条confirmStatus==CS03, 且只有一条, 返回本条记录
    6. 如最新一条confirmStatus==CS03, 且有多条, 返回最近的confirmStatus==CS02的记录

字段取数说明:

  1. channelDesc: 根据channelId >> customer_channel_id 返回 customer_channel_text
  2. currMoCount: 到表9中统计月份t_month==当前月的结果集中所有的result == "RS03", 结果集按检查日期check_date去重统计次数
  3. lastMoCount: 到表9中统计月份t_month==上月的结果集中所有的result == "RS03", 结果集按检查日期check_date去重统计次数
  4. execDate: 到表9中按相应的agreement_id找最新的一条检查日期check_date
  5. tpName:根据promotionId >> trade_promotion_id 返回 promotion_name
  6. distributorName: 根据distributorId >> bussiness_partner_no 返回 customer_name
  7. granted: 表15中限制+经销商代垫状态??? 条件待确认
  8. grantedQuickPay: ??? 条件待确认
  9. distributorName: 根据distributorId >> bussiness_partner_no 返回 customer_name
  10. confirmStatus:
    1. 同一agreementId中最新的一条的confirmStatus
    2. 如confirmStatus==CS04, 即修改待确认, 本条记录不返还, 返回倒数第二条confirmStatus==CS01的协议, 并将返回的倒数第二条协议的confirmStatus填充CS04

  3. CPM_PROMOT_RETURN_SKU_HEADER (促销返还记录表-实物-抬头表)
  4. CPM_PROMOT_RETURN_SKU_LINER (促销返还记录表-实物-明细表)

接口逻辑:

  1. 卡券 (rebatePattern == RT03):
    1. 主键agreementId, month
    2. 到表1中使用主键查找, 如果命中返还status=0001, msg=该月已返还; 否则继续
    3. 调用卡券中心接口/card/deposit 发放卡券, 该接口返还包含卡券code的列表
    4. 将步骤3中返回的卡券code列表结合入参 插入表1
  2. 现金 (rebatePattern == RT02):
    1. 主键agreementId, month
    2. 到表2中使用主键查找, 如果命中返还status=0001, msg=该月已返还; 否则继续
    3. 将入参数据插入表2
  3. 实物 (rebatePattern == RT01):
    1. 主键agreementId, month
    2. 到表3中使用主键查找, 如果命中, 根据命中的produt_rebate_id到表4中查找后, 一并删除表3 表4中的记录
    3. 将入参插入表3 表4
