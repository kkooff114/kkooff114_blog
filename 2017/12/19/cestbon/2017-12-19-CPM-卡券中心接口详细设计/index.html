CPM 卡券中心接口详细设计

1. 创建卡券

/card/create

入参:

字段 描述
operator 调用者用户名 
cardType 卡券类型:WATER水; BEVERAGE饮料 
title 卡券标题 
notice 提示 
description 描述 
leastCost 启用金额, 单位分, 0无门槛 
validType 卡券有效期类型FIX_TIME_RANGE(固定日期区间), FIX_TERM(固定时长, 自领取后按天算), 这里写死FIX_TERM 
fixed_term valid_type为FIX_TERM时专用, 卡券领取多少天内有效 
begin_timestamp valid_type为FIX_TIME_RANGE时专用, 有效开始时间 
end_timestamp valid_type为FIX_TIME_RANGE时专用, 有效结束时间 
get_limit 每人获得卡券上限 
use_limit 每人使用卡券上限 
promotion_id 卡券关联业务值, 如01034343, T-100003 

出参:

字段 描述
cardId 卡券id

说明:

入参直接入库card_base_info 卡券基础信息表 并返回card_id

2. 发放卡券

/card/deposit

入参:

字段 描述
operator 调用者用户名
promotionId 活动id 
agreementId 协议id 
memberId 用户id 
reduceCost 总金额 

出参:

字段 描述
cardCode 卡券code

说明:

  1. 用入参的promotionId到表 card_base_info中查找并带出相应的card_id.
  2. 将入参reduceCost按最大面额100切分成多条记录插入card_inventory 卡券状态表; 规则: 如果传出328, 则切分成三条100,100,28; 如果传入98, 则无需切分.
  3. 将promotionId, agreementId, 及步骤2中产生的card_code插入card_relationship_tp 卡券业务关联表-活动
  4. 将memberId, 及步骤2中产生的card_code插入card_relationship_member 卡券业务关联表-用户
