对自己做了三年的一个app的总结

发表于 2017-03-07   |   分类于 Android   |     |   阅读次数   |  

背景

稍后整理…

注意点

一定要和使用者及时沟通

他们才是能够真正反馈有价值信息的人, 找几个有责任心, 热爱工作的使用者, 和他们做朋友.

需要搜集使用信息

需要搜集的信息有很多, 业务类型的, 操作类型的, bug类型的, 日志可以很好的追溯操作者每天的使用情况, 帮助解决一些疑难杂症, 也可弥补因你的失误造成业务数据丢失

帮助中心

一般企业的app都会有培训, 亦或是通过其他老同事的培训, 但是如果增加帮助中心, 是锦上添花的事情.

反馈与建议

可以更大范围了搜集使用者的建议, 帮助改善应用

字体大小调整后,界面的适配

如果你的用户包含40+岁的使用者, 就一定要注意这点, 他们会有调大字体的需求

需要让用户知道每次都更新了哪些内容

让他们期待每次的更新

应用的稳定性高于一切

尤其是这种企业级的应用, 业务每天都使用app进行工作, app的稳定性就直接决定了每天工作的心情, 甚至是他们的收入. 其次是易用性.
